# LTP RFC Template

> Use this template for any protocol or governance change. Keep it concise, additive, and explicit about compatibility.

## Summary
- One paragraph describing the change and who it serves.

## Motivation
- Problem statement and why the current protocol/governance is insufficient.

## Scope
- **In-scope:**
- **Out-of-scope:**

## Backwards Compatibility
- Describe how existing nodes/SDKs remain interoperable.
- State any required feature negotiation or default behaviors.

## Wire Format Impact
- Frames, fields, or types affected (additive only).
- Encoding/decoding changes and defaults.

## Conformance Impact
- Tests/fixtures required and expected outcomes.
- Tooling or harness updates.

## Security Considerations
- Threats, mitigations, and any disclosure handling needs.

## Reference Implementation Notes (optional)
- Implementation hints or sample code locations.

## Alternatives Considered
- Brief list of other approaches and why they were rejected.

## Rollout Plan
- Migration steps, interoperability testing, and exit criteria.
