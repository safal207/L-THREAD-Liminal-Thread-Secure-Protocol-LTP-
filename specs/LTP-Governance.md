# LTP Governance (Draft)

## Scope
This document defines how the Liminal Thread Protocol evolves.

## Principles
- Core stability over feature velocity
- Backward compatibility by default
- Explicit non-goals
- Small frozen core, extensible edges

## Roles
- Maintainer (Core Steward)
- Contributors
- Implementers (SDK / Node authors)
- Observers (users, integrators)

## Decision Making
- Core changes require RFC + explicit approval
- Extensions evolve independently
- No breaking changes to frozen Core versions

## Versioning
- Core versions are frozen
- Extensions may iterate freely
