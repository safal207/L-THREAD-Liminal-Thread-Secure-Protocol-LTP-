# LTP RFC Process

## What is an RFC?
A Request for Change or Clarification to the LTP specification.

## When an RFC is required
- Any change to Core
- Any semantic change to Canonical Flow
- Any change affecting conformance

## RFC Lifecycle
1. Draft
2. Discussion
3. Accepted / Rejected
4. Implemented
5. Frozen (if Core)

## RFC Format
- Motivation
- Proposal
- Alternatives
- Compatibility
- Security considerations
